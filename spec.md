# 文件上传下载校验管理系统 - 产品规格文档

## 1. 产品概述

### 1.1 产品定位
一个面向数据导入场景的文件管理系统，支持 Excel、CSV 格式文件的上传、校验、存储、查询和下载。

### 1.2 目标用户
- 普通用户：上传文件、下载文件、查看详情
- 管理员：所有用户权限 + 删除文件

### 1.3 使用场景
- 用户按模板上传数据文件
- 系统自动校验数据格式和必填项
- 校验通过后存储数据，失败则返回具体错误原因
- 支持文件查询、详情预览、下载、删除

## 2. 功能需求

### 2.1 用户认证与授权

| 功能点 | 描述 |
|--------|------|
| 用户注册 | 页面注册，密码支持数字和英文 |
| 管理员创建用户 | 管理员可创建用户 |
| 用户登录 | 登录后2小时超时，支持多地同时登录 |
| 权限控制 | 删除权限仅管理员可用 |

### 2.2 字段配置

| 功能点 | 描述 |
|--------|------|
| 固定字段 | 序号、身份证、手机号（必填） |
| 自定义字段 | 用户可配置最多10个自定义字段 |
| 字段类型 | 文本（最长64）、数字（整数）、日期（YYYY-MM-DD） |
| 必填设置 | 自定义字段可设置为必填或选填 |
| 配置生效 | 配置后需重新下载模板才能生效 |
| 历史兼容 | 字段配置变更不影响历史数据 |

### 2.3 模板管理

| 功能点 | 描述 |
|--------|------|
| 模板生成 | 固定字段 + 用户配置的自定义字段 |
| 模板下载 | 用户下载当前配置的Excel/CSV模板 |
| 列顺序 | 严格按照固定字段+自定义字段顺序 |
| 列名校验 | 上传时严格校验列名 |
| 单元格格式 | Excel和CSV模板所有单元格均为文本格式，避免数字被自动转换（如身份证号、手机号等长数字） |

### 2.4 文件上传

| 功能点 | 描述 |
|--------|------|
| 支持格式 | Excel (.xlsx, .xls)、CSV (.csv) |
| 文件大小 | 最大200MB |
| 并发上传 | 支持同时上传多个文件 |
| 文件命名 | 系统自动命名：用户名_日期时间_第几个文件 |
| Sheet处理 | 仅读取第一个Sheet |
| 空行处理 | 跳过空行 |
| 空格处理 | 字段前后自动trim |
| 表头处理 | 第一行为表头，数据从第二行开始，表头不校验 |

### 2.5 数据校验

#### 校验规则

| 字段 | 校验规则 |
|------|----------|
| 序号 | 必填 |
| 身份证 | 必填，18位格式+校验码验证 |
| 手机号 | 必填，11位数字且1开头 |
| 文本类型 | 最长64字符 |
| 数字类型 | 整数 |
| 日期类型 | YYYY-MM-DD格式 |
| 自定义必填字段 | 根据配置校验 |
| 同文件重复 | 同一文件内身份证号不能重复 |

#### 校验失败处理

| 处理方式 | 描述 |
|----------|------|
| 上传策略 | 校验失败整文件拒绝上传 |
| 错误提示 | 返回具体失败原因（行号+字段+错误信息） |
| 数据存储 | 校验失败不存储任何数据 |

## 4. 业务流程

### 4.1 用户首次使用流程

```
1. 用户注册/管理员创建用户
2. 用户登录
3. 用户配置自定义字段（可选）
4. 用户下载模板
5. 用户按模板填写数据
6. 用户上传文件
7. 系统校验
   - 成功：存储数据，列表展示
   - 失败：返回错误信息，不存储数据
```

### 4.2 修改字段配置流程

```
1. 用户进入字段配置页面
2. 修改自定义字段配置
3. 保存配置
4. 重新下载模板
5. 按新模板填写数据
6. 上传文件（历史数据不受影响）
```

### 4.3 数据校验失败流程

```
1. 用户上传文件
2. 系统逐行校验
3. 发现错误行时立即中断
4. 返回错误信息：行号、字段、错误原因
5. 用户修改文件
6. 重新上传
```

### 5.4 数据明细表 (t_data_detail)

**设计说明：采用混合存储方案（固定字段 + JSON字段）**

- **固定字段**：序号、身份证、手机号独立存储，支持高效索引、查询、去重校验
- **自定义字段**：使用 JSON 字段存储，灵活扩展，支持动态字段配置
- **优势**：核心字段高效查询 + 自定义字段灵活扩展 + 节省存储空间

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键 |
| file_id | BIGINT | 文件ID，索引 |
| seq_no | VARCHAR(32) | 序号 |
| id_card | VARCHAR(32) | 身份证号，索引（用于去重校验） |
| phone | VARCHAR(16) | 手机号 |
| custom_fields | JSON | 自定义字段JSON，格式见下方 |
| row_num | INT | 原文件行号 |
| create_time | DATETIME | 创建时间 |

**custom_fields JSON 格式示例：**

```json
{
  "姓名": "张三",
  "地址": "北京市朝阳区",
  "邮箱": "zhangsan@example.com",
  "入职日期": "2024-01-01",
  "年龄": "25"
}
```

**JSON 字段查询示例（MySQL 5.7+）：**

```sql
-- 查询自定义字段
SELECT id, seq_no, id_card, phone,
       JSON_EXTRACT(custom_fields, '$.姓名') AS name,
       JSON_EXTRACT(custom_fields, '$.地址') AS address
FROM t_data_detail
WHERE file_id = 1;

-- 按自定义字段筛选
SELECT * FROM t_data_detail
WHERE JSON_EXTRACT(custom_fields, '$.姓名') = '张三';
```

## 9. 附录

### 9.1 数据存储设计说明

#### 设计方案：混合存储（固定字段 + JSON字段）

| 方案 | 固定字段 | 自定义字段 |
|------|----------|------------|
| **字段存储** | 序号、身份证、手机号 → 独立列 | 其他字段 → JSON列 |
| **索引支持** | 支持高效索引 | 通过JSON函数查询 |
| **扩展性** | 固定 | 灵活扩展，无需改表结构 |
| **适用场景** | 核心查询字段、去重校验 | 动态配置字段 |

#### 字段配置表与数据表的关系

```
t_field_config (字段配置)
├── user_id: 1
├── field_name: "姓名", field_type: "TEXT", sort_order: 1
├── field_name: "地址", field_type: "TEXT", sort_order: 2
└── field_name: "入职日期", field_type: "DATE", sort_order: 3

           ↓ 用户上传文件后

t_data_detail (数据存储)
├── seq_no: "1"
├── id_card: "110101199001011234"
├── phone: "13800138000"
└── custom_fields: {
      "姓名": "张三",
      "地址": "北京市朝阳区",
      "入职日期": "2024-01-01"
    }
```

#### 历史数据兼容性处理

**核心机制：字段配置快照**

当用户上传文件时，系统将当前的字段配置保存为快照到 `t_file_record.field_config_snapshot`

```
用户A的字段配置演变：
├── 2024-01-01: [姓名, 地址, 邮箱]           → 上传文件1，保存快照1
├── 2024-01-15: [姓名, 地址, 邮箱, 年龄]     → 修改配置
└── 2024-01-20: [姓名, 地址, 手机, 入职日期] → 修改配置，上传文件2，保存快照2

文件展示时：
├── 文件1：使用快照1 [姓名, 地址, 邮箱] 解析JSON
└── 文件2：使用快照2 [姓名, 地址, 手机, 入职日期] 解析JSON
```
